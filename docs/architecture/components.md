# Component Architecture for Clothing Store (MVP)

Цей документ описує основні компоненти системи інтернет-магазину одягу, їх призначення та технологічний стек.

| Назва компонента | Призначення | Основні функції | Інтерфейс/Протокол | Дані (вхід/вихід) | Технології |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Store Frontend** | Веб-інтерфейс для користувачів | • Каталог товарів<br>• Кошик та оформлення замовлення<br>• Особистий кабінет | HTTP/HTTPS | Вхід: Дії користувача<br>Вихід: HTML/JSON сторінки | React / Next.js |
| **Core Backend API** | Основна бізнес-логіка | • Управління товарами (CRUD)<br>• Обробка замовлень<br>• Авторизація користувачів | REST API / JSON | Вхід: JSON запити<br>Вихід: JSON відповіді | Node.js (NestJS) або Python (Django) |
| **Database (Primary)** | Зберігання структурованих даних | • Зберігання користувачів<br>• Інформація про товари та склад<br>• Історія замовлень | SQL (TCP/IP) | Вхід: Дані про замовлення<br>Вихід: Вибірки товарів | PostgreSQL |
| **Image Storage** | Файлове сховище медіа | • Зберігання фотографій одягу | HTTP / S3 API | Вхід: Файли зображень<br>Вихід: URL посилання | AWS S3 / MinIO |
| **Payment Gateway** | Зовнішній сервіс оплати | • Обробка платежів картками | HTTPS (Webhook) | Вхід: Сума, дані картки<br>Вихід: Статус оплати | Stripe / LiqPay |
| **Delivery API** | Зовнішній сервіс доставки | • Розрахунок вартості доставки<br>• Вибір відділення | HTTPS (REST) | Вхід: Місто, вага<br>Вихід: Список відділень, ціна | Nova Poshta API |
